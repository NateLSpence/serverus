Serverus Help

Prerequisites

Set up WordPress and log in

You'll need a host and domain name with an updated version of WordPress installed in order to use Serverus.

Your web host might have an option to install WordPress automatically, or you may have to download and install it yourself.

Once you have WordPress up and running, log in.

As administrator, you'll see the WordPress Admin Toolbar at the top of the webpage- other users won't see this. You'll also be able to see IP addresses and extra buttons on posts for moderation. Blocked/muted users and posts will be visible to you, but not regular users. Otherwise, what you see is what your users will see for the most part.

Installation

Theme Installation

Go to the Admin page and navigate to Appearance->Themes. Click "Add New" at the top of the page.

Click "Upload Theme" at the top of the page.

Click "Choose File" and navigate to the serverus-theme.zip file. Upload it, and click "Install Now".

Now activate the theme.

Theme Activation

Once you click the "Activate" button, you should be taken to a screen that will run you through some first-time setup.

This screen is always available under Appearance->Theme Activation.

Once you're done selecting your options, click "Save Changes".

Plugins

Required and Recommended Plugins

Upon activating Serverus, you will see a nagging box asking you to install some plugins. Click "Begin installing plugins".

If you've dismissed the prompt, you can still view these plugins under "Appearance->Install Plugins".

Select the plugins you would like to install. See below for an explanation of each plugin's functions.

Select "Install" under the "Bulk Actions" dropdown and hit Apply.

You'll need to enter your FTP credentials. You'll find these somewhere on your web host's dashboard.

Once the plugins finish downloading and installing, you have one more step! Configure the Easy Bootstrap Shortcodes plugin to work with Serverus by going to "EBS Settings" on the left, selecting the "Use from theme..." option for the JS and CSS settings, and clicking "Update Settings".

If you didn't install the Easy Bootstrap Shortcodes plugin, you're clear! You can return to the Dashboard and continue to set up your site.

Required and Recommended Plugins

bbPress is the forum software. It's required for Serverus to run.

bbPress Moderation makes it easier to prevent spam by making a user's first post require approval by a moderator.

bbPress Signature adds the ability for users to put a signature in their posts.

Easy Bootstrap Shortcodes gives you more options for pretty formatting when making additional pages.

Author Avatars List lets you display a widget showing all of your forum's users. Common use would be to set the Avatar size to 32 or 16, and show the following roles: Keymaster, Moderator, and Participant.

bbPress Admin Bar Addition gives Admins and Moderators a handy menu on the top Admin bar to better manage the forums.

Simple Custom CSS adds a place for you to put in your own CSS (under "Appearance").

Add Custom Scripts lets you put in your own Javascript (like Google Analytics, for example). Found under "Tools">"Header/Footer Scripts".


Feel free to browse around for more plugins! If there's something you want to do with your site or forum, there's usually a plugin that makes it easy.

Setup

General Wordpress Setup

You should check out the Settings pages and decide what will be best for your site. The most important ones to look at are General, Reading, Forums, and bbPress Moderation.

Note that "Writing" and "Discussion" aren't very useful to you, since those default WordPress functions aren't in use with Serverus. Also, anything referencing blog posts/comments are defunct. Serverus is for forums, and removes the default blogging platform.

Front Page Setup

Like most forums, you can have your front page display news. This will display the latest topics from a single forum (for example, "Server News"), newest first, without any of the replies attached to them.

Edit the front page to adjust these options.

Serverus uses a shortcode to determine which forum to get news from, and to set some options:

[srv_frontpage forum_id=# posts_per_page=5 char_limit=0 show_stickies=false]

Firstly, forum_id=# is the ID of the forum to display. Browse to a forum to find its ID (only Admins and Mods see them). Use a number here; no quotes or symbols.

Then, posts_per_page=5 decides how many posts to display on the front page before you have to click "Next".

The option char_limit=0 is the maximum length, in characters, to display for each post. 0 means there is no limit.

Lastly, show_stickies=false decides whether or not to display "sticky" threads in your news page. Use true or false here.


An example of a customized shortcode might look like this:

[srv_frontpage forum_id=12 posts_per_page=20 char_limit=450 show_stickies=false]

In this example, our front page will display posts from the forum with the ID of 12. It will display a maximum of 20 posts, but only the first 450 characters from each. Sticky threads won't be displayed.


Remember to click "Update" when you're finished editing the page.

Forums Setup

To create a forum, click "New Forum" in the "Forums" menu.

First create your forum's categories. All forums require a category, even if everything is under the same category.

Next, create the forums you want and place them under their categories.

Check your forums page to make sure everything is displaying correctly.

Menus

This is no different from your typical WordPress menus. If your pages aren't showing up in your Nav bar and you're not sure how to add them, read this guide. To change the colors of the Nav bar, check out the "Customizing" section below.

Widgets


Recent Topics Widget

This widget displays a list of recent posts made by your users.

Other Widgets

The (bbPress) Forums Statistics widget and (bbPress) Forums List widgets might be useful as well. Text widgets work for any static information you want visible (Server IP, links, etc). For any third-party widgets, paste the code into a Text widget.

Advertisements

There are three places you can display ads for revenue.

The first is in the sidebar, the second is on the front page (where the front page shortcode is).

And the third is directly below the first post in a thread.

To use these, paste the code for your ad in a Text widget. The location of the widget (Primary, Frontpage, Under First Post) will determine where the ad displays.

Users

Enable Registration

To open your forums to the world, you'll have to enable registration in order for users to make their own account. Make sure "Anyone can register" is checked under Settings.

Alternatively, you can opt to add users manually via the Users menu.

Modify Users

There are two places you can edit a user's information. The first is the Users menu from the WordPress menus.

The second is accessed by clicking on the user's name in the forums and viewing their profile.

These are basically the same, but as an administrator you might find them both useful. In the WordPress menus, you can see all of the users in a list, instead of digging through posts to find a specific person. Making quick tweaks might be easier from the forum profile view.

Moderation

If you've installed the bbPress Moderation plugin, then a user's first post won't be displayed until it's approved. Only administators and moderators can see and approve them.

You can mark it as spam on the Replies menu.

Or you can select it and click "Publish" to approve it.

In the forum, individual posts have buttons for moderation.

From either a user's profile view or from the "Users" menu, you can set a user to "Blocked" as their forum role to ban them. Setting a user to "Moderator" gives them the ability to edit other users/posts and see a minimal version of the WordPress administration menus.

Customizing

The "Customize" screen

Time to customize! Access the Customize screen from one of three places:

Add a header image and text, a background image, and choose a color scheme that suits your fancy.

Serverus is simple, but you can dig into the color scheme if you want to. The best method is to choose a color scheme that is closest to what you need already, and then tweak it from there.

If you're itching to change something, but it doesn't have an option, you can always override the default CSS, assuming you've installed the Simple CSS plugin.

Single pages

If you have the Easy Bootstrap Shortcodes plugin installed, you'll see buttons for Bootstrap elements when editing a page.

Make a Landing Page/Splash Screen

To make a page without the Nav bar and sidebar, choose the template labeled "Splash".

Keep in mind that since this disables the Nav bar, so you'll need to include a link so users can click through to the rest of your site.

To make this page the home page, go to "Settings" -> "Reading", and under "Front Page Displays", select the dropdown box labeled "Front Page" and choose your Splash page.

Disable the Sidebar

Same as above, but choose the template labeled "No Sidebar".

Troubleshooting

I can't get to any of my pages.

If you're getting an error along the lines of "404 page not found" then you probably need to enable Pretty Permalinks and mod_rewrite. If you're having trouble with this, contact your web host's support.

I just see a white screen!

This is what WordPress does when it breaks. What is the very last thing you did before you saw the white screen? If you just enabled the theme, or one of the plugins, your best bet is to follow WordPress's help guide. After that, double-check that you've updated WordPress, bbPress, and Serverus itself to their latest versions. Double-check the Prerequisites section. Lastly, remember that bbPress must be activated for Serverus to work properly. Disabling it while Serverus is activated might cause issues.

When I add a widget, it collapses and I can't see the options.

Click the "Save" button after you add the widget, and it should expand again. If it doesn't, try refreshing the page.

Dropdown menus aren't working.

Most likely, this is being caused by the Easy Bootstrap Shortcodes plugin. If you haven't configured it yet, check out the directions for configuring it and make sure you've done this.

This problem is usually because the dropdown.js plugin in Bootstrap is being included twice. Serverus is built on Bootstrap, so it doesn't need bootstrap.js included again. If the above directions don't fix your problem, see if there's another reason you might have bootstrap.js (which comes with dropdown.js in it) being included a second time.

There's a third-party plugin I want to use, but it doesn't work right with Serverus.

I can help you with that! Post it in Serverus's support section and I'll check it out. Make sure to tell me what specifically isn't working right with the plugin for you.

Everything else.

First, try to determine where your problem is occurring. Is it your web host, WordPress, a specific plugin, or the theme itself? Don't be shy to ask for help in Serverus's support section, but be aware that I'll only be useful for theme-related issues.
